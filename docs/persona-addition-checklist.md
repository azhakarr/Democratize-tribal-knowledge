# Persona Addition Checklist

Use this checklist when adding or updating personas in docs/octoacme-roles-and-personas.md.

- [ ] Role name and short Role Summary added
- [ ] Responsibilities listed (3–6 bullets)
- [ ] Typical Interactions / Handoffs documented (who they work with and why)
- [ ] Acceptance criteria for role ownership clarified (e.g., who leads releases, who owns security sign-off)
- [ ] Link to any related playbooks / runbooks (release, incident, security)
- [ ] Update table of contents (if present)
- [ ] Reference associated issue (e.g., closes #5) in the PR description
- [ ] Request reviews from impacted stakeholders (PM, Engineering Lead, Security, SRE, Support)

Why: Ensures new roles are introduced consistently and reduce ambiguity on ownership and handoffs.
