# OctoAcme Project Management Docs

This README indexes the OctoAcme program management process documents and provides a short summary of our core project management processes so contributors and new team members can quickly find guidance.

## Brief summary of OctoAcme project management processes

- **Initiation**: Capture the project one-pager (problem, objective, success metrics), identify stakeholders, and decide whether to move into planning.
- **Planning**: Create a prioritized backlog with acceptance criteria, estimate scope, define Definition of Done, and map releases/milestones.
- **Execution & Tracking**: Use a project board (Backlog → Ready → In Progress → In Review → QA → Done), follow PR and CI conventions, run tests, and track velocity and blockers.
- **Release & Deployment**: Follow pre-release checks (CI, security scans, release notes), run smoke tests, and use rollback/incident playbooks when necessary.
- **Retrospective & Continuous Improvement**: Run retros after milestones, convert action items into backlog issues, and measure the impact of improvements.
